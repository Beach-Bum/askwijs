UX Research Findings
What Builds Trust in Finance Apps
Synthesized from 2024–2025 fintech UX research — applied specifically to a ZZP tax AI context.
🧭

Progressive Disclosure

Show only what's needed now. ZZPers are busy — don't dump all tax info at once. Reveal complexity on demand.

High impact
💬

Conversational Interface

40% of fintech apps now use AI advisors. A chat-first UI feels like asking a knowledgeable friend — perfect for the ZZP audience.

Core feature
📊

Dashboard = Calm Overview

Show netto earnings, BTW owed, and next deadline upfront. Users trust apps that show them control, not chaos.

Trust signal
📅

Deadline Visibility

Dutch ZZPers fear Belastingdienst penalties. Proactive deadline reminders reduce anxiety and build loyalty.

Anxiety reducer
✅

Micro-interactions

Subtle animations on button presses, saved states, and completed filings signal that the system is working — humanizes the AI.

Delight layer
🌍

Bilingual by Default

Dutch-native
Design Direction
Do's & Don'ts
Boundaries that keep the brand feeling trustworthy and human.

✓ Do

  • Use generous white space — signals clarity
  • Show progress bars in multi-step flows
  • Use green for confirmations and success states
  • Keep the chat response tone warm and direct
  • Use Dutch tax terms correctly and confidently
  • Personalize the greeting with the user's name
  • Make security visible (lock icon, SSL badge)
  • Round corners on cards — feels approachable

✗ Don't

  • Don't use red outside of error/alert states
  • Don't show raw tax numbers without context
  • Don't use corporate banking aesthetics — too cold
  • Don't use heavy gradients or flashy animations
  • Don't bury T&C's — show key points upfront
  • Don't use dark mode as default (less trust for finance)
  • Don't use jargon without a plain-language follow-up
  • Don't use serif fonts below 16px — illegible
